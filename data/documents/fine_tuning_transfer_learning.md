# Fine-Tuning and Transfer Learning

Fine-tuning and transfer learning are techniques for adapting pretrained models to specific tasks or domains, dramatically reducing the data and compute required compared to training from scratch.

## Transfer Learning Fundamentals

Transfer learning leverages knowledge gained from one task (pretraining) to improve performance on a different but related task (downstream). The key insight is that early layers of neural networks learn general features (edges, syntax patterns, semantic relationships) that transfer across tasks, while later layers learn task-specific features.

The standard workflow is:
1. Start with a model pretrained on a large dataset (ImageNet for vision, internet text for language)
2. Replace or add task-specific output layers
3. Fine-tune on the target dataset, often with a lower learning rate

Transfer learning works because it provides better weight initialization than random initialization, enables learning from small datasets, and reduces training time by orders of magnitude.

## Full Fine-Tuning

Full fine-tuning updates all parameters of a pretrained model on a new dataset. This gives maximum flexibility but requires significant compute and risks catastrophic forgetting — where the model loses its general capabilities while specializing.

Guidelines for full fine-tuning:
- Use a learning rate 10-100x smaller than pretraining (typically 1e-5 to 5e-5 for LLMs)
- Use warmup steps (5-10% of total steps) followed by linear or cosine decay
- Monitor validation loss for overfitting, especially with small datasets
- Keep training short — 1-5 epochs is usually sufficient

## Parameter-Efficient Fine-Tuning (PEFT)

PEFT methods update only a small subset of parameters, keeping most of the pretrained model frozen. This dramatically reduces compute, memory, and storage requirements while often matching full fine-tuning performance.

### LoRA (Low-Rank Adaptation)

LoRA is the most popular PEFT method. Instead of modifying weight matrices W directly, it learns low-rank decomposition matrices A and B such that the modified weight is W + BA, where B has shape (d, r) and A has shape (r, d) with rank r << d.

Key advantages:
- Reduces trainable parameters by 100-1000x (e.g., from 7B to 7M)
- Multiple LoRA adapters can be swapped at inference time without changing the base model
- Training requires 3-10x less GPU memory than full fine-tuning
- Typical rank values: r = 4, 8, 16, or 32

### QLoRA (Quantized LoRA)

QLoRA combines 4-bit quantization of the base model with LoRA fine-tuning. This enables fine-tuning a 65B parameter model on a single 48GB GPU. The base model is quantized to 4-bit NormalFloat (NF4) precision, while LoRA adapters are trained in bfloat16.

### Prefix Tuning and Prompt Tuning

Prefix tuning prepends learnable continuous vectors ("virtual tokens") to the input at every layer, while prompt tuning only prepends them at the embedding layer. Both methods freeze the entire model and only learn the prefix/prompt embeddings.

Prompt tuning becomes competitive with full fine-tuning as model size increases. At 10B+ parameters, prompt tuning with 100 virtual tokens can match full fine-tuning performance.

### Adapters

Adapter layers are small neural network modules inserted between existing layers of a pretrained model. Each adapter typically consists of a down-projection, nonlinearity, and up-projection, creating a bottleneck architecture. Only adapter parameters are trained; everything else is frozen.

## Instruction Fine-Tuning

Instruction fine-tuning trains a base LLM on a dataset of (instruction, response) pairs to follow human instructions. This is distinct from pretraining (next-token prediction on raw text) and is what transforms a base model into a helpful assistant.

Key datasets for instruction tuning:
- **FLAN Collection**: 1800+ NLP tasks formatted as instructions
- **Alpaca**: 52K instruction-following examples generated by GPT-4
- **Dolly**: 15K human-written instruction-response pairs
- **OpenAssistant**: Multi-turn conversational data

The quality of instruction data matters far more than quantity. 1000 high-quality, diverse examples can outperform 100K noisy examples. Data should cover diverse tasks, formats, and difficulty levels.

## Reinforcement Learning from Human Feedback (RLHF)

RLHF aligns model outputs with human preferences through three stages:
1. **Supervised Fine-Tuning (SFT)**: Fine-tune on high-quality demonstration data
2. **Reward Model Training**: Train a model to predict human preferences between output pairs
3. **PPO Optimization**: Use Proximal Policy Optimization to maximize the reward model's score while staying close to the SFT model (KL divergence penalty)

RLHF is computationally expensive and unstable. Alternatives include:
- **DPO (Direct Preference Optimization)**: Eliminates the reward model by directly optimizing on preference data. Simpler, more stable, and competitive with RLHF.
- **ORPO (Odds Ratio Preference Optimization)**: Combines SFT and alignment into a single training stage.
- **KTO (Kahneman-Tversky Optimization)**: Works with binary (good/bad) feedback instead of pairwise preferences.

## Domain Adaptation

Adapting models to specific domains (medical, legal, financial, scientific) typically involves:
1. **Continued pretraining** on domain-specific text corpus
2. **Instruction fine-tuning** on domain-specific tasks
3. **Evaluation** on domain-specific benchmarks

Examples:
- **Med-PaLM**: Google's medical model, continued pretraining on medical literature
- **BloombergGPT**: Financial domain model trained on financial documents
- **CodeLlama**: Meta's code-specialized model, continued pretraining on code

## Data Preparation for Fine-Tuning

Data quality is the most important factor in fine-tuning success:
- **Diversity**: Cover the range of expected inputs
- **Quality**: Each example should represent ideal behavior
- **Format consistency**: Standardize input-output format
- **Decontamination**: Remove evaluation benchmark data to prevent leakage
- **Balanced distribution**: Avoid overrepresenting any category

Typical dataset sizes:
- Classification tasks: 100-1000 examples per class
- Instruction following: 1000-50000 diverse examples
- Domain adaptation pretraining: 1M-1B tokens of domain text

## Evaluation After Fine-Tuning

Always evaluate fine-tuned models against:
- **Task-specific metrics**: Accuracy, F1, BLEU, ROUGE depending on the task
- **General capability benchmarks**: MMLU, HellaSwag, ARC to check for capability regression
- **Safety benchmarks**: TruthfulQA, toxicity evaluation to ensure alignment is maintained
- **Human evaluation**: Expert assessment of output quality, particularly for open-ended generation tasks

## Catastrophic Forgetting

Fine-tuning on a narrow task can cause the model to forget previously learned knowledge. Mitigation strategies include:
- Using a very low learning rate
- Mixing general-purpose data with task-specific data during fine-tuning
- PEFT methods (LoRA, adapters) that preserve most original weights
- Elastic Weight Consolidation (EWC) that penalizes changes to important weights
- Progressive fine-tuning starting from the last layers
